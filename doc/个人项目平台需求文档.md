# 个人项目平台 需求文档

- 文档版本：v1.0
- 编写日期：2026-07-14
- 文档状态：草稿（待评审）

---

## 1. 背景与目标

### 1.1 背景
个人在日常工作与生活中经常需要处理多项待办事项，容易遗忘要做的事情、忘记截止时间。市面上的项目管理平台（如参考的 KLD 项目平台）功能强大但偏重团队协作、流程复杂，个人使用成本高。因此需要一个**轻量、简洁、以「不遗忘」为核心**的个人项目管理平台。

### 1.2 目标
1. 能够创建任务、设置任务时间（计划开始 / 截止时间）；任务默认归属于当前用户本人（个人平台不做「指派给他人」）。
2. 能够对任务进行「开始 / 暂停 / 完成」等状态操作。
3. 提供工作台视图，一进平台就能看到「指派给我」的待办，起到提醒、防遗忘作用。
4. 提供最基础的人员管理：注册、登录（仅校验密码），并有一个超级管理员账号可管理已注册人员。
5. **后端采用 API 优先架构**，为后续开发安卓客户端共享同一套数据、执行相同操作打下基础。

### 1.3 非目标（本期不做）
- 不做复杂的审批流、工时统计、燃尽图等团队协作重功能。
- 不做邮箱验证码 / 短信验证 / 第三方登录（登录仅校验用户名 + 密码）。
- 不做任务的多人协同评论、附件版本管理等（可作为后续迭代）。
- 安卓客户端本期只做接口预留，不实现具体 App。

## 2. 用户角色

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| 超级管理员（Super Admin） | 系统内置的最高权限账号，通常是平台拥有者本人 | 拥有普通用户全部权限 + 人员管理（查看/启用/禁用/重置密码/设置角色） |
| 普通用户（User） | 通过注册产生的普通账号 | 项目与任务的创建、指派、状态操作；只能管理自己可见的数据；**无人员管理入口** |

> 说明：系统初始化时自动创建一个超级管理员账号（如 `admin`），首次部署后应立即修改其密码。

## 3. 功能需求

### 3.1 用户认证模块

#### 3.1.1 注册
- 输入项：用户名、密码、确认密码。
- 校验规则：
  - 用户名唯一，不可重复；长度 2–20 个字符。
  - 密码长度 ≥ 6 位；密码与确认密码需一致。
- 注册成功后默认角色为「普通用户」，状态为「启用」。
- 密码在后端以哈希（推荐 bcrypt）存储，禁止明文保存。

#### 3.1.2 登录
- 输入项：用户名、密码。
- 仅校验用户名与密码是否匹配（无验证码）。
- 登录成功后由后端签发访问令牌（推荐 JWT），前端与安卓端后续请求携带该令牌。
- 登录失败给出统一提示「用户名或密码错误」。
- 被禁用的账号无法登录，提示「账号已被禁用，请联系管理员」。

#### 3.1.3 退出登录
- 清除本地令牌，返回登录页。

### 3.2 人员管理模块（仅超级管理员）

- 成员列表：展示用户名、角色、注册时间、状态（启用/禁用）、最近登录时间。
- 支持按用户名搜索。
- 操作：
  - **重置密码**：管理员将某用户密码重置为指定的新密码（或系统默认密码）。
  - **启用 / 禁用**：禁用后该用户不能登录。
  - **设置角色**：将普通用户提升为管理员或降回普通用户。
  - （可选）**删除用户**：软删除，保留其历史任务数据。
- 普通用户登录后，导航栏不显示「成员管理」入口，直接访问接口也应被后端拦截（403）。

### 3.3 项目模块

- 项目是「项目 → 多个任务」的层级归类，一个项目下可包含多个任务（个人使用可只建一个「默认项目」）。
- 项目字段：项目名称、项目编号、负责人、排序号、项目描述、状态（进行中/已完成/已归档）。
- 功能：创建项目、编辑项目、项目列表（展示任务总数、未完成任务数）、按名称/状态筛选。
- 新建任务时可选择所属项目；若不选择则归入「默认项目」。

### 3.4 任务模块（核心）

#### 3.4.1 任务字段
| 字段 | 必填 | 说明 |
|------|------|------|
| 任务名称 | 是 | 简短标题 |
| 所属项目 | 否 | 默认为「默认项目」，体现项目→任务的层级 |
| 任务类型 | 否 | 如 开发 / 修复 / 测试 / 调研 / 杂务，可预置枚举 |
| 优先级 | 否 | 高 / 中 / 低（默认中） |
| 计划开始时间 | 否 | 日期时间 |
| 截止时间 | 否 | 日期时间，用于逾期提醒 |
| 任务描述 | 否 | 富文本或纯文本 |
| 创建人 | 系统 | 自动记录 |
| 完成人 / 完成时间 | 系统 | 完成时自动记录 |

#### 3.4.2 任务状态机
```
未开始 ──开始──▶ 进行中 ──完成──▶ 已完成
  │                 │  ▲
  │              暂停│  │继续
  │                 ▼  │
  └──取消─────▶ 已暂停 ─┘
（未开始/进行中/已暂停 均可「取消」为 已取消；已完成可「重开」回到 进行中）
```
| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| 未开始 | 新建默认状态 | 开始、取消、编辑 |
| 进行中 | 正在处理 | 暂停、完成、取消、编辑 |
| 已暂停 | 临时挂起 | 继续（回到进行中）、完成、取消、编辑 |
| 已完成 | 已办结 | 重开、查看 |
| 已取消 | 作废 | 查看 |

#### 3.4.3 任务操作
- 创建任务、编辑任务、删除任务（软删除）。
- 一键状态操作：开始 / 暂停 / 继续 / 完成 / 取消 / 重开（在列表行内以图标按钮提供）。
- 任务列表筛选：按状态、类型、所属项目、时间范围、关键字。

#### 3.4.4 图片与附件
- 任务描述可以很长（大文本），编辑区高度充足；新建/编辑任务弹窗做大以便录入。
- **图片**：支持上传图片并保存到服务器，两种方式：
  1. 点击图片上传区选择本地图片（支持多张、`image/*`）。
  2. 在描述框内 **Ctrl+V 直接粘贴截图**。
  展示为缩略图，可点击查看大图、可删除。
- **附件**：支持上传**任意类型文件**（文档、压缩包、表格等），显示文件名与大小，可下载、可删除。
- 上传后由后端保存文件到服务器（本地磁盘目录或对象存储），数据库仅记录访问路径/URL。
- 限制：单个文件大小上限（图片建议 ≤ 10MB，附件建议 ≤ 50MB，可配置）；存储目录按 `年/月` 或任务 ID 分目录，避免单目录文件过多。

### 3.5 工作台模块（首页）

- 用户登录后默认进入工作台，聚焦「我的待办」，起到防遗忘作用。
- 视图 Tab：
  - **待办**：当前用户未开始 / 进行中的任务（默认页）。
  - **已暂停**：当前用户已暂停的任务。
  - **已完成**：当前用户已完成的任务。
  - **全部**：当前用户的全部任务。
- 逾期任务（截止时间早于当前时间且未完成）截止时间标红高亮。
- （后续迭代）可增加「今天到期」「本周到期」的快速统计卡片。

## 4. 非功能需求

- **架构：API 优先**。后端提供无状态 REST API（JSON），Web 前端与未来的安卓 App 共用同一套接口与数据库。
- **安全**：密码哈希存储；接口基于令牌鉴权；人员管理等接口做服务端角色校验。
- **响应性能**：常规列表接口响应 < 500ms（数据量在个人使用规模下）。
- **易用性**：界面简洁，参考主流项目平台的表格 + 弹窗交互习惯。
- **可维护性**：前后端分离，模块清晰，便于后续扩展安卓端。

## 5. 技术选型建议

| 层 | 建议方案 | 备注 |
|----|----------|------|
| 后端 | Spring Boot（Java） 或 NestJS（Node.js） | 提供 REST API + JWT 鉴权 |
| 数据库 | MySQL 8 / PostgreSQL | 关系型，满足任务/人员/项目建模 |
| Web 前端 | Vue 3 + Element-Plus | 与参考平台观感一致，表格/弹窗组件成熟 |
| 鉴权 | JWT（Bearer Token） | 无状态，方便安卓端复用 |
| 图片存储 | 服务器本地磁盘目录 或 MinIO/OSS 对象存储 | 数据库存路径，文件存服务器；安卓端通过 URL 访问同一份图片 |
| 安卓端（后续） | 原生 Kotlin 或 Flutter | 直接调用同一套 REST API |

## 6. 数据模型（概要）

### user（用户）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| username | varchar | 用户名，唯一 |
| password_hash | varchar | 密码哈希 |
| role | enum | ADMIN / USER |
| status | enum | ENABLED / DISABLED |
| created_at | datetime | 注册时间 |
| last_login_at | datetime | 最近登录时间 |

### project（项目）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| name | varchar | 项目名称 |
| code | varchar | 项目编号 |
| owner_id | bigint FK→user | 负责人 |
| sort_no | int | 排序号 |
| description | text | 描述 |
| status | enum | ACTIVE / DONE / ARCHIVED |
| created_at | datetime | 创建时间 |

### task（任务）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| title | varchar | 任务名称 |
| project_id | bigint FK→project | 所属项目 |
| owner_id | bigint FK→user | 归属用户（即创建人本人） |
| type | varchar | 任务类型 |
| priority | enum | HIGH / MEDIUM / LOW |
| status | enum | TODO / DOING / PAUSED / DONE / CANCELED |
| plan_start_at | datetime | 计划开始时间 |
| due_at | datetime | 截止时间 |
| description | text | 任务描述（支持内嵌图片） |
| finished_at | datetime | 完成时间 |
| created_at | datetime | 创建时间 |

### attachment（图片 / 附件）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| task_id | bigint FK→task | 所属任务 |
| kind | enum | IMAGE / FILE（图片或普通附件） |
| file_name | varchar | 原始文件名 |
| file_path | varchar | 服务器存储相对路径 |
| url | varchar | 访问 URL |
| size | bigint | 文件大小（字节） |
| mime_type | varchar | 如 image/png、application/pdf |
| uploaded_at | datetime | 上传时间 |

## 7. 接口设计（概要）

> 均以 `/api` 为前缀，返回统一 JSON 结构 `{ code, message, data }`。

### 认证
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/auth/register | 注册 |
| POST | /api/auth/login | 登录，返回 token |
| POST | /api/auth/logout | 退出 |
| GET  | /api/auth/me | 获取当前用户信息 |

### 人员管理（管理员）
| 方法 | 路径 | 说明 |
|------|------|------|
| GET  | /api/admin/users | 用户列表 |
| PUT  | /api/admin/users/{id}/status | 启用/禁用 |
| PUT  | /api/admin/users/{id}/role | 设置角色 |
| PUT  | /api/admin/users/{id}/password | 重置密码 |

### 项目
| 方法 | 路径 | 说明 |
|------|------|------|
| GET  | /api/projects | 项目列表 |
| POST | /api/projects | 创建项目 |
| PUT  | /api/projects/{id} | 编辑项目 |

### 任务
| 方法 | 路径 | 说明 |
|------|------|------|
| GET  | /api/tasks | 任务列表（支持筛选参数） |
| POST | /api/tasks | 创建任务 |
| PUT  | /api/tasks/{id} | 编辑任务 |
| DELETE | /api/tasks/{id} | 删除任务（软删除） |
| PATCH | /api/tasks/{id}/status | 变更状态（start/pause/resume/finish/cancel/reopen） |
| GET  | /api/workbench/tasks | 工作台任务（按 todo/paused/done/all 分组，均为当前用户） |

### 图片 / 附件
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/attachments | 上传图片或文件（multipart/form-data，含 kind），返回 url + id |
| GET  | /api/tasks/{id}/attachments | 获取某任务的图片与附件列表 |
| DELETE | /api/attachments/{id} | 删除图片/附件（同时删除服务器文件） |

## 9. 后续迭代规划

1. **安卓客户端**：基于本文档的 REST API 开发，实现登录、工作台、任务操作，与 Web 端共享数据。
2. **到期提醒**：本地/推送通知，任务到期或逾期时提醒。
3. **数据统计**：今日/本周待办统计卡片、完成率图表。
4. **富文本描述与附件**：任务描述支持富文本、图片、附件上传。

## 附录：与参考平台（KLD 项目平台）的差异

| 维度 | KLD 项目平台 | 本平台 |
|------|-------------|--------|
| 定位 | 团队协作、研发流程管理 | 个人待办与项目管理 |
| 登录 | 企业账号体系 | 仅用户名+密码 |
| 任务字段 | P/D/Q 多维度、需求关联、Bug 关联 | 精简为必要字段 |
| 人员管理 | 完整组织架构 | 最简：注册/登录 + 超管管理 |
| 客户端 | Web | Web + 规划安卓端（共享 API） |
